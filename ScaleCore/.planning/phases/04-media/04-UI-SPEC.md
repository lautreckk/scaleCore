# Phase 4 — UI Design Contract

> Visual and interaction contract for the Media Library UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## UI Scope

Phase 4 has two domains: inbound media processing (MEDIA-01/02/03) and media library (LIB-01/02/03/04/05). Only LIB-01 and LIB-02 require new UI. The rest is backend pipeline work.

**New UI surface:** A "Biblioteca de Midia" (Media Library) section within the existing agent form, visible only in edit mode. This section allows uploading, listing, editing, and deleting media files associated with an agent.

**No new pages.** The media library lives inside the existing agent edit page as a new section below the existing form fields (after escalation keywords, before the save button).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 0%)` / --background | Page background, form background |
| Secondary (30%) | `hsl(0 0% 4%)` / --card | Media item cards, upload drop zone background |
| Accent (10%) | `hsl(0 84% 50%)` / --primary | Upload button, active media indicator, file type badges |
| Destructive | `hsl(0 84% 50%)` / --destructive | Delete media button, remove confirmation |
| Muted | `hsl(0 0% 15%)` / --muted | Borders, disabled states, drop zone border |
| Muted foreground | `hsl(0 0% 64%)` / --muted-foreground | Placeholder text, secondary metadata text |

Accent reserved for: Upload CTA button, active/enabled toggle indicator on media items, file type icon badge backgrounds.

**Destructive interaction contract:** Since `--primary` and `--destructive` share the same red hue (`0 84% 50%`), destructive actions must be distinguished from the accent CTA through context (icon + copy), not color alone. The delete button uses `ghost` variant at rest (no background fill). On hover and focus, the delete button applies `bg-destructive/15` (15% opacity destructive background) with `border border-destructive/40` to create a lighter, outlined treatment that visually separates it from the solid-fill upload CTA. The upload CTA uses the `default` variant with full solid `bg-primary` fill.

**Source:** Detected from `globals.css` CSS custom properties. This is a dark-first theme with red accent.

## Interaction Contract

### Visual Focal Point

**Primary visual anchor:** The "Adicionar midia" button in the section header row. This is the main call-to-action and should be the first element the user's eye lands on within the media library section.

### Upload Flow

1. User clicks "Adicionar midia" button (outline variant, Upload icon from lucide)
2. Native file picker opens (accept: image/*, video/*, audio/*, .pdf, .doc, .docx)
3. File validation runs client-side:
   - Max file size: 10MB
   - Accepted types: image (jpeg, png, webp, gif), video (mp4, webm), audio (mp3, ogg, wav), document (pdf, doc, docx)
4. On valid file: upload starts immediately, progress shown via Loader2 spinner on the button
5. After upload: new media item appears at top of list with empty name/description fields in edit mode
6. User fills name (required) and description (required), clicks "Salvar" inline button
7. On save: fields become read-only, item shows in display mode
8. On upload error: sonner toast with error message, no item created

### Media Item Display

Each media item card shows:
- **Left:** Thumbnail (64x64px, object-cover, rounded-md) for images; file type icon (lucide: Image, Video, Music, FileText) for non-previewable types
- **Center:** Name (14px semibold, single line, truncate), Description (12px regular muted-foreground, 2 lines max, truncate), Media type badge (12px, secondary variant), File size (12px muted-foreground)
- **Right:** Edit button (ghost, Pencil icon, `aria-label="Editar {name}"`), Delete button (ghost, Trash2 icon, `aria-label="Remover {name}"`, destructive hover/focus style per Color section)

### Edit Inline

1. User clicks Pencil icon on a media item
2. Name and description become editable Input fields (pre-filled)
3. Two inline buttons appear: "Salvar" (default variant, sm size), "Cancelar" (ghost variant, sm size)
4. On save: PATCH request, fields revert to display mode, sonner success toast
5. On cancel: fields revert to display mode, no changes

### Delete Flow

1. User clicks Trash2 icon on a media item
2. AlertDialog opens with confirmation copy (see Copywriting section)
3. On confirm: DELETE request, item removed from list with no animation, sonner success toast
4. On cancel: dialog closes, no action

### Empty State

When agent has no media items, show centered empty state within the media library section area (not full page).

### Loading State

While fetching media list: show 3 Skeleton cards (same dimensions as MediaItemCard) stacked vertically.

### Limits

- Maximum 20 media items per agent (enforced by API, shown in UI as "{count}/20 midias" counter next to section heading)
- When limit reached: upload button disabled with tooltip "Limite de 20 midias atingido"

## Layout

### Media Library Section (within agent form, edit mode only)

```
[Separator]
[Section heading: "Biblioteca de Midia"  |  "3/20 midias"  |  [Adicionar midia] button]
[ScrollArea max-h-[400px]]
  [MediaItemCard]  -- stacked vertically, gap-8px
  [MediaItemCard]
  [MediaItemCard]
[/ScrollArea]
```

- Section appears after escalation keywords section, before the form submit button
- Hidden in create mode (agent must be saved first to have an ID for media association)
- ScrollArea with max-height 400px to prevent the form from becoming excessively long

### MediaItemCard Layout

```
[Thumbnail 64x64] [gap-16px] [Name + Description + Meta row] [gap-auto] [Edit btn] [Delete btn]
```

- Horizontal flex layout
- Card with `bg-card` background, `border-border` border, `rounded-lg`, `p-4` (16px padding)
- Meta row: Badge(media type) + file size text, separated by 8px gap

## Accessibility Notes

- Upload button must have `aria-label="Adicionar midia ao agente"`
- Edit icon-only buttons must have `aria-label="Editar {name}"` where `{name}` is the media item name
- Delete icon-only buttons must have `aria-label="Remover {name}"` where `{name}` is the media item name
- Media item cards must be keyboard-navigable (Tab to focus, Enter to edit)
- Delete confirmation dialog auto-focuses the cancel button (safe default)
- File type badges use text labels, not color-only differentiation
- Thumbnail images must have `alt` attribute set to media name
- Disabled upload button (at limit) must have `aria-disabled="true"` and tooltip via `title` attribute
