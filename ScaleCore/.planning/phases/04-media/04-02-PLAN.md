1. Create `app/api/agents/[id]/media/route.ts` with GET and POST handlers:

**GET /api/agents/[id]/media** — List all media for an agent:
- Auth: `supabase.auth.getUser()` -> 401 if no user
- Tenant: `tenant_users` lookup -> 404 if no tenant
- Verify agent ownership: `supabase.from("ai_agents").select("id").eq("id", agentId).eq("tenant_id", tenantUser.tenant_id).single()` -> 404 if not found
- Query: `supabase.from("ai_agent_media").select("*").eq("agent_id", agentId).order("created_at", { ascending: false })`
- Return: `{ data: AiAgentMediaRow[] }`

**POST /api/agents/[id]/media** — Upload new media:
- Auth + tenant + agent ownership (same as GET)
- Count check: `supabase.from("ai_agent_media").select("id", { count: "exact" }).eq("agent_id", agentId)` -> if count >= 20, return `{ error: "Limite de 20 midias atingido" }` with status 400
- Parse FormData: `file` (File), `name` (string), `description` (string)
- Validate file size: max 10MB (10 * 1024 * 1024) -> 400 if exceeded
- Validate file type: accept image/jpeg, image/png, image/webp, image/gif, video/mp4, video/webm, audio/mp3, audio/ogg, audio/wav, application/pdf, application/msword, application/vnd.openxmlformats-officedocument.wordprocessingml.document
- Determine media_type from MIME: image/* -> "image", video/* -> "video", audio/* -> "audio", application/* -> "document"
- Upload to Supabase Storage: bucket `chat-media`, path `agent-media/${agentId}/${timestamp}-${random}.${ext}`
- Insert into `ai_agent_media` table: `{ agent_id, name, description, media_type, file_url, mime_type, file_size }`
- Return: `{ data: AiAgentMediaRow }` with status 201

2. Create `app/api/agents/[id]/media/[mediaId]/route.ts` with PATCH and DELETE handlers:

**PATCH /api/agents/[id]/media/[mediaId]** — Update media name/description:
- Auth + tenant + agent ownership
- Parse JSON body: `{ name?: string, description?: string }`
- Validate: name must be non-empty string if provided
- Update: `supabase.from("ai_agent_media").update({ name, description, updated_at: new Date().toISOString() }).eq("id", mediaId).eq("agent_id", agentId).select().single()`
- Return: `{ data: AiAgentMediaRow }`

**DELETE /api/agents/[id]/media/[mediaId]** — Delete media:
- Auth + tenant + agent ownership
- Fetch media record to get file_url for storage cleanup
- Delete from table: `supabase.from("ai_agent_media").delete().eq("id", mediaId).eq("agent_id", agentId)`
- Delete from storage: extract path from file_url, `supabase.storage.from("chat-media").remove([path])`
- Return: `{ success: true }` with status 200

Both route files use `{ params }: { params: Promise<{ id: string }> }` and `{ params }: { params: Promise<{ id: string; mediaId: string }> }` patterns matching Next.js 15 async params convention used in existing routes.
