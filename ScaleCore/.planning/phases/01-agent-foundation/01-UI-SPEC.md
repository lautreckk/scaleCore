## Color

All colors use the existing HSL CSS custom properties defined in `app/globals.css`. This is a dark-only theme.

| Role | CSS Variable | HSL Value | Usage |
|------|-------------|-----------|-------|
| Dominant (60%) | `--background` | `0 0% 0%` (pure black) | Page background, main surfaces |
| Secondary (30%) | `--card` / `--secondary` | `0 0% 4%` / `0 0% 10%` | Agent cards, form containers, sidebar |
| Accent (10%) | `--primary` | `0 84% 50%` (crimson red) | See reserved list below |
| Destructive | `--destructive` | `0 84% 50%` (same as primary) | Delete agent button, destructive confirmations |
| Muted text | `--muted-foreground` | `0 0% 64%` | Secondary descriptions, helper text |
| Border | `--border` | `0 0% 15%` | Card borders, input borders, separators |

Accent reserved for:
- "Criar Agente" primary CTA button
- Active/inactive toggle when active (Switch component)
- Focus ring on inputs (`--ring`)
- Tag preview badge when valid
- Primary action in AlertDialog ("Confirmar" button in bulk tag dialog)

## Page & Component Contracts

### 1. Agent Listing Page (`/agentes`)

**Layout:** Page header with title + CTA button, followed by responsive card grid.

- Header: `text-2xl font-bold text-white` with `Bot` icon from lucide-react, "Criar Agente" button aligned right
- Grid: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4` (16px gaps)
- Empty state: Centered vertically in content area, `Bot` icon at 48px, heading + body + CTA button
- Loading state: 6 Skeleton cards matching card dimensions

### 2. Agent Card (`agent-card.tsx`)

**Structure:**
- Card with `--card` background, `--border` border, hover state with subtle brightness increase
- Top row: Agent name (`text-lg font-medium text-white`) + Switch toggle (active/inactive) aligned right
- Middle row: Model name as Badge (`text-xs`), tag as Badge with outline variant
- Bottom row: Instance count (`text-sm text-muted-foreground`) + status indicator
- Click anywhere on card (except Switch) navigates to `/agentes/[id]`

**States:**
- Active: Switch on, card at normal opacity
- Inactive: Switch off, card at 60% opacity, "Inativo" badge in muted color
- Loading: Skeleton with matching dimensions

### 3. Agent Form Page (`/agentes/novo` and `/agentes/[id]`)

**Layout:** Single-column form, max-width 672px (42rem), centered.

**Form sections (top to bottom, separated by `Separator`):**

1. **Informacoes Basicas**
   - Nome do Agente: `Input` (required, max 100 chars)
   - Prompt do Sistema: `Textarea` (required, max 10000 chars, min-height 200px)

2. **Modelo LLM**
   - Modelo: `Select` dropdown. Each option displays: model name, provider tag, credit cost ("~N creditos/msg")

3. **Tag de Ativacao**
   - Tag: `Input` with real-time slug preview below (`Badge` showing formatted tag)
   - Helper text: "Apenas letras minusculas, numeros e hifens"
   - Apply mode: `RadioGroup` with "Apenas novos chats" (default) and "Todos os chats existentes"

4. **Instancias WhatsApp**
   - Checkbox list inside `ScrollArea` (max-height 240px if more than 4 instances)
   - Each row: `Checkbox` + instance name + phone number + status dot (green=connected, gray=disconnected)
   - If no instances: muted message with link to settings

5. **Acoes**
   - Create: "Salvar Agente" primary button
   - Edit: "Salvar Agente" primary button + "Excluir Agente" destructive button (outline variant, right-aligned)
   - Cancel: "Cancelar" ghost button, navigates back to `/agentes`

**States:**
- Loading (edit page): Skeleton for entire form
- Submitting: Button disabled with loading spinner, "Salvando..."
- Validation errors: Red border on field + `FormMessage` below field with error text
- Success: Toast notification + redirect to `/agentes`

### 4. Delete Confirmation (`delete-dialog.tsx`)

- Uses `AlertDialog` component
- Title: "Excluir Agente"
- Body: Includes agent name interpolated
- Cancel: secondary variant
- Confirm: destructive variant ("Excluir")

### 5. Bulk Tag Dialog (`bulk-tag-dialog.tsx`)

- Uses `AlertDialog` component
- Triggered when user selects "Todos os chats existentes" and saves
- Shows count of affected chats (fetched from API before showing dialog)
- Cancel: secondary variant
- Confirm: primary variant ("Aplicar Tag")

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Card click | Navigate to `/agentes/[id]` edit page |
| Card Switch toggle | Toggle `is_active` via PATCH API, optimistic update, toast on success/error |
| Tag input typing | Real-time slugify preview below input field (debounce not needed, pure transform) |
| Tag duplicate detection | On form submit, API returns 409 if tag exists, display inline error on tag field |
| Bulk tag confirm | Only shown when `tag_apply_mode` = "all_existing" AND form is being saved. Fetch count first, then show dialog. Apply on confirm. |
| Delete agent | AlertDialog confirmation required. On confirm, DELETE API call, toast, redirect to `/agentes` |
| Form cancel | Navigate back to `/agentes` without saving. No unsaved-changes warning (matches existing project patterns). |
| Instance checkbox | Immediate visual feedback. Persisted only on form save. |
