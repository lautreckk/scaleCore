1. Create `app/api/agents/route.ts` with GET and POST:

**GET** — List all agents for tenant:
- Auth flow: `createClient()` -> `getUser()` -> `tenant_users` lookup (exact same pattern as warming/configs)
- Query: `supabase.from("ai_agents").select("*, ai_agent_instances(*, whatsapp_instances:instance_id(id, name, phone_number, status))").eq("tenant_id", tenantUser.tenant_id).order("created_at", { ascending: false })`
- Return: `NextResponse.json({ data: agents })`

**POST** — Create new agent:
- Auth + tenant resolution (same pattern)
- Parse body, validate with `agentFormSchema.parse(body)` (import from `@/lib/agents/validation`)
- Insert into `ai_agents`: `{ tenant_id: tenantUser.tenant_id, name, system_prompt, model_id, activation_tag, tag_apply_mode, is_active }`
- If instance_ids provided, insert into `ai_agent_instances` for each: `{ agent_id: newAgent.id, instance_id }`
- Handle unique constraint violation (Postgres error code `23505` on `unique_tag_per_tenant`): return `NextResponse.json({ error: "Essa tag ja esta sendo usada por outro agente. Escolha uma tag diferente." }, { status: 409 })`
- Return: `NextResponse.json({ data: newAgent }, { status: 201 })`

2. Create `app/api/agents/[id]/route.ts` with GET, PATCH, DELETE:

**Params handling:** Use `{ params }: { params: Promise<{ id: string }> }` pattern — `const { id } = await params;` (Next.js 14.2 async params pattern, matching warming/configs/[id])

**GET** — Single agent:
- Auth + tenant resolution
- Query: `supabase.from("ai_agents").select("*, ai_agent_instances(*, whatsapp_instances:instance_id(id, name, phone_number, status))").eq("id", id).eq("tenant_id", tenantUser.tenant_id).single()`
- Return 404 if not found

**PATCH** — Update agent:
- Auth + tenant resolution
- Parse body, validate partial fields
- Update `ai_agents` row: `supabase.from("ai_agents").update({ ...fields, updated_at: new Date().toISOString() }).eq("id", id).eq("tenant_id", tenantUser.tenant_id)`
- If `instance_ids` in body: delete all existing `ai_agent_instances` for this agent, then insert new ones (replace strategy)
- Handle `23505` unique violation on tag same as POST
- Return updated agent

**DELETE** — Delete agent:
- Auth + tenant resolution
- Delete: `supabase.from("ai_agents").delete().eq("id", id).eq("tenant_id", tenantUser.tenant_id)`
- Junction table rows cascade-deleted automatically
- Return `NextResponse.json({ success: true })`

3. Create `app/api/agents/[id]/tags/route.ts` with POST:

**POST** — Bulk apply tag to existing chats:
- Auth + tenant resolution
- Get agent: verify it belongs to tenant, get `activation_tag` and instance_ids from `ai_agent_instances`
- First, count affected chats (for response): `supabase.from("chats").select("id", { count: "exact", head: true }).in("instance_id", instanceIds).not("tags", "cs", `{${agent.activation_tag}}`)`
- Then apply via RPC: `supabase.rpc("apply_agent_tag", { p_instance_ids: instanceIds, p_tag: agent.activation_tag })`
- Return `NextResponse.json({ affected: count })`

Add a GET endpoint on the same route to fetch the count of chats that would be affected (used by the bulk tag dialog before confirmation):
- Same auth + tenant + agent resolution
- Count chats without the tag in the agent's instances
- Return `NextResponse.json({ count })`

**Note on AGENT-07 scope:** This bulk-apply endpoint handles retroactive tag application only. Auto-tagging new incoming chats is Phase 2 webhook logic and is NOT implemented here.
